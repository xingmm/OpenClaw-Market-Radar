# 市场洞察编排技能

## 适用触发词
- `跑一下市场洞察`
- `跑一下投资雷达`
- `给我今天的合并市场洞察`

## 核心职责
- 先做宏观洞察，再做快讯雷达，最后合并为单份市场洞察。
- 输出必须包含宏观定价层、市场结构层、投资执行层。

## 输入要求
- 必需输入：宏观数据 JSON + 快讯雷达 JSON。
- 可选输入：持仓与偏好文件、上一期投资雷达。

## 执行步骤
1. 调用宏观洞察能力（P19）。
2. 调用快讯雷达能力（P20）。
3. 做冲突信号归一化并给出主导判断。
4. 输出合并报告（P21）与行动清单。

## 输出格式
1. 宏观定价层
2. 市场结构层
3. 投资执行层（含持仓映射）
4. 行动清单（3项）与证伪条件（3条）

## 质量门禁
- 严禁跳过“宏观 -> 快讯 -> 合并”任一步骤。
- 冲突信号必须先解释冲突再给结论。
- 缺任一必填章节视为失败输出。

## 失败处理
- 缺少任一输入数据时，输出“无法生成合并洞察”并列明缺口。
- 置信度不足时，降级为“观察版结论”并给触发条件。

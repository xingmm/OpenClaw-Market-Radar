# P19: 宏观洞察（Macro Insight）

## 适用场景

用于单独输出宏观层结论。当用户触发“跑一下宏观洞察”或“宏观怎么看”且要求系统化输出时使用。

## 输入

- `macro_json`: `scripts/fetch_macro_liquidity.py` 产出的最新 JSON。
- `optional_news_json`: 可选，市场快讯数据（仅用于交叉验证，不替代宏观主结论）。

## 处理流程

1. 先写数据完整性头：报告期、数据源文件名、抓取时间、缺失字段。
2. 只用宏观变量做判断，不用指数涨跌替代宏观结论。
3. 对关键变量给出当前值 + 5日变化 + 20日变化（低频数据用“本期 vs 上期”）。
4. 每条结论必须给出“证据数据 -> 推理链 -> 结论”。

## 强制覆盖维度

1. 全球宏观温度计（美国流动性/中国流动性/黄金油价与美元利率）
2. 宏观边际变化看板
   - 美债利率
   - 美元指数或美元强弱
   - 美元兑人民币
   - 黄金
   - 原油
   - 中国流动性六件套（有数据则必须）
3. 流动性传导判断
   - 宏观变量 -> 折现率/风险溢价 -> 资产风格 -> A股主线

## 输出格式

```markdown
# 宏观洞察 - YYYY-MM-DD

## 数据完整性头
- 报告期：
- 数据源：
- 抓取时间：
- 缺失字段：

## 1) 全球宏观温度计

## 2) 宏观边际变化看板

## 3) 流动性传导判断

## 4) 宏观结论（含证据链）
- 结论A：证据 -> 推理 -> 结论
- 结论B：证据 -> 推理 -> 结论
```

## 质量门禁

- 缺任何强制维度即判定失败。
- 结论若无证据链即判定失败。
- 严禁只输出观点，不给数据依据。

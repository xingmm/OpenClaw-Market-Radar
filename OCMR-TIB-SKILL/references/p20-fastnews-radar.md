# P20: 快讯雷达（Fastnews Radar）

## 适用场景

用于单独输出快讯层结论。当用户触发“跑一下快讯雷达”或要求事件优先级排序时使用。

## 输入

- `fastnews_json`: `scripts/fetch_fastnews_portfolio.py` 的输出 JSON。
- `holding_md`: `OpenClaw/我的持仓与关注点和投资偏好/我的持仓.md`。
- `optional_macro_json`: 可选，用于校验事件是否有宏观共振。

## 处理流程

1. 去重：同主题重复标题合并。
2. 分桶：至少输出 `持仓相关` 和 `宏观/市场` 两个桶（有数据时）。
3. 打分：`relevance + impact + freshness`。
4. 输出 top-k，并给每条事件写“事实 -> 传导 -> 结论”。

## 输出格式

```markdown
# 快讯雷达 - YYYY-MM-DD

## 数据完整性头
- 数据源：
- 抓取时间：
- 样本数量：

## 1) 持仓相关
- 事件：
  - 事实：
  - 传导：
  - 结论：
  - 标签：利好/利空/中性，短/中/长

## 2) 宏观/市场
- 事件：
  - 事实：
  - 传导：
  - 结论：

## 3) 优先级清单
- P1:
- P2:
- P3:
```

## 质量门禁

- top-k 结果中不得出现重复标题。
- 结论必须可追溯到事件事实。
- 输出必须给出优先级标签（P1/P2/P3）。

# P21: 市场洞察编排器（Market Insight Orchestrator）

## 适用场景

用于“跑一下市场洞察”合并输出。该提示词负责把宏观洞察与快讯雷达整合成一份用户最终报告。

## 依赖输入

- `macro_report`: 来自 P19 的宏观洞察结果。
- `fastnews_report`: 来自 P20 的快讯雷达结果。
- `holding_profile`: 持仓与偏好文件（可选但建议）。

## 编排原则

1. 分层输出，不混层：
   - 宏观定价层
   - 市场结构层
   - 投资执行层
2. 如宏观与快讯信号冲突：
   - 先给冲突点，再给主导判断与证伪条件。
3. 每个核心结论必须满足：
   - 证据数据 -> 推理链 -> 最终结论。

## 输出格式

```markdown
# 市场洞察 - YYYY-MM-DD

## 数据完整性头
- 报告期：
- 输入来源：P19/P20 文件与时间戳
- 缺失字段：

## 1) 宏观定价层

## 2) 市场结构层（板块/主题/事件催化）

## 3) 投资执行层（全A股映射 + 个人持仓映射）

## 4) 行动清单
- Top 3 actions
- Top 3 risks
- Top 3 watch triggers
- Next review timestamp

## 5) 三条证伪条件
- 条件1:
- 条件2:
- 条件3:
```

## 质量门禁

- 缺任一层即失败。
- 缺行动清单或证伪条件即失败。
- 严禁只给摘要而无分层正文（除非用户明确要简版）。

# 快讯雷达技能

## 适用触发词
- `跑一下快讯雷达`
- `看看有什么信号`
- `今天快讯里有什么重点`

## 核心职责
- 对快讯做去重、分桶、打分、优先级排序。
- 至少输出 `持仓相关` 与 `宏观/市场` 两类结果（有数据时）。
- 强制增加 `重要信息清单`（逐条列出时间、事件、影响），禁止只给总结。
- 人工智能相关主线不能忽略：`AI大模型/机器人/GPU/算力/半导体` 必须单列检查。
- 新能源相关主线不能忽略：`新能源/光伏/储能/汽车（含新能源车）` 必须单列检查。

## 输入要求
- 必需输入：`fetch_fastnews_portfolio.py` 输出 JSON。
- 可选输入：持仓文件、关注板块偏好。

## 执行步骤
1. 读取快讯并做标题去重。
2. 进行主题分桶（持仓相关/宏观市场）。
3. 按相关性、影响力、时效性打分。
4. 生成 P1/P2/P3 优先级清单与动作建议。

## 输出格式
1. 重要信息清单（至少10条，逐条写明“时间 + 事件 + 一句话影响”）
2. 今日重点信号（P1）
3. 监控信号（P2）
4. 背景信号（P3）
5. 证据事件与影响说明

## 参考资料
- 主参考：`../../p20-fastnews-radar.md`
- 执行标准：`../../../../RUNBOOK.md`

## 质量门禁
- 必须先去重再排序。
- 每条结论必须对应具体事件事实。
- 需给出优先级及对应动作建议。
- 必须有“重要信息清单”，且不能只写抽象的“冲突链新增”。
- 必须检查并说明AI主线是否有增量（大模型/机器人/GPU/算力/半导体）。
- 必须检查并说明新能源主线是否有增量（新能源/光伏/储能/汽车）。

## 失败处理
- 快讯为空时，输出“无有效快讯输入”。
- 持仓文件缺失时，降级为市场快讯模式并明确提示。
